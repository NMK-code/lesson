Архитектура WPF: XAML, Code-behind, Application

Разберем структуру WPF-приложений и взаимодействие XAML с кодом на C#

Цель лекции

Изучить архитектуру WPF-приложений, понять роли XAML, Code-behind и App.xaml, научиться связывать интерфейс и логику приложения, а также познакомиться с управлением ресурсами и событиями.

Основной материал

1. XAML

XAML (Extensible Application Markup Language) — декларативный язык разметки, который описывает интерфейс приложения. XAML позволяет:

  • Создавать визуальные элементы (кнопки, текстовые поля, списки)
  • Устанавливать свойства элементов (цвет, размер, шрифты)
  • Определять расположение элементов (панели, контейнеры)
  • Привязывать данные и события
Пример XAML (MainWindow.xaml)
<Window x:Class="WpfApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Пример WPF" Height="250" Width="400">
    <Grid Margin="10">
        <TextBlock Text="Введите имя:" Margin="0,0,0,10" />
        <TextBox Name="textBoxName" Width="200" />
        <Button Content="Сохранить" Click="Save_Click" Margin="0,10,0,0"/>
    </Grid>
</Window>

Обратите внимание на атрибут x:Class, который связывает XAML с C# кодом.

Совет: Разделяйте визуальную разметку и логику приложения — это упрощает поддержку и масштабирование проекта.

2. Code-behind

Code-behind — это файл на C#, связанный с XAML, где реализуется логика работы интерфейса и обработка событий. Файл имеет ту же структуру имени, что и XAML, но с расширением .xaml.cs.

Пример C# (MainWindow.xaml.cs)
using System.Windows;

namespace WpfApp
{
    public partial class MainWindow : Window
    {
        public MainWindow()
        {
            InitializeComponent(); // Инициализация компонентов, описанных в XAML
        }

        private void Save_Click(object sender, RoutedEventArgs e)
        {
            string name = textBoxName.Text;
            MessageBox.Show($"Привет, {name}!", "Событие Click");
        }
    }
}

Метод InitializeComponent() создается автоматически и связывает XAML элементы с объектами C#.

Примечание: Любые изменения интерфейса, заданные в XAML, автоматически доступны в Code-behind через Name элементов.

3. App.xaml и App.xaml.cs

App.xaml управляет глобальными ресурсами приложения и точкой входа. Обычно здесь задаются стили, темы и запуск главного окна.

Пример App.xaml
<Application x:Class="WpfApp.App"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             StartupUri="MainWindow.xaml">
    <Application.Resources>
        <SolidColorBrush x:Key="PrimaryColor" Color="#4b2e83"/>
        <Style TargetType="Button">
            <Setter Property="Background" Value="{StaticResource PrimaryColor}" />
            <Setter Property="Foreground" Value="White" />
            <Setter Property="Padding" Value="5,2" />
        </Style>
    </Application.Resources>
</Application>

App.xaml.cs содержит метод Main(), который создается автоматически и запускает приложение.

4. Взаимодействие компонентов

Иерархия взаимодействия компонентов WPF выглядит так:

  • XAML: описание интерфейса и визуальных компонентов
  • Code-behind: обработка событий и логика взаимодействия с элементами
  • App.xaml: глобальные ресурсы, стили и точка входа приложения
Пример поиска элемента через Code-behind
// Найдем TextBox по имени и изменим текст
TextBox tb = this.FindName("textBoxName") as TextBox;
if(tb != null)
{
    tb.Text = "Привет, WPF!";
}

5. Дополнительные возможности

  • Ресурсы — цвета, стили, шаблоны можно хранить в App.xaml и использовать во всех окнах
  • События — Routed Events позволяют элементам реагировать на действия пользователя, даже если события поднимаются через дерево элементов
  • DataContext — связывает интерфейс с данными или ViewModel (важно для паттерна MVVM)
Совет: Использование ресурсов и стилей повышает консистентность интерфейса и упрощает поддержку приложения.

6. Сравнение с Windows Forms

Характеристика Windows Forms WPF
Разделение UI и логики Смешанный код и интерфейс Полное разделение через XAML и Code-behind
Стили и ресурсы Ограниченные возможности Поддержка глобальных ресурсов, шаблонов, тем
Графика Ограниченная 2D графика Поддержка 2D/3D, анимаций, медиаконтента
Масштабируемость Сложно поддерживать DPI Поддержка масштабируемых и векторных элементов
Примечание: Понимание архитектуры WPF помогает создавать более гибкие, масштабируемые и поддерживаемые приложения.
