Основной материал

Что такое WPF?

Windows Presentation Foundation (WPF) — это технология Microsoft для создания современных настольных приложений с богатым пользовательским интерфейсом. WPF позволяет создавать графически насыщенные приложения с поддержкой 2D/3D графики, анимаций и медиаконтента.

Преимущества WPF

  • Разделение логики и интерфейса через XAML и Code-behind
  • Поддержка масштабируемой графики и DPI
  • Богатый набор элементов управления и стилей
  • Возможность создания сложных визуальных эффектов и анимаций
  • Интеграция с другими технологиями .NET

Структура WPF-приложения

Типичное WPF-приложение состоит из следующих компонентов:

  • App.xaml — главный файл приложения, содержит глобальные ресурсы и настройки.
  • MainWindow.xaml — основной интерфейс приложения, написанный на XAML.
  • Code-behind — C# код, обрабатывающий события и логику взаимодействия.
Пример XAML (MainWindow.xaml)
<Window x:Class="WpfApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Пример WPF" Height="250" Width="400">
    <Grid>
        <Button Content="Нажми меня" Width="120" Height="40" HorizontalAlignment="Center" VerticalAlignment="Center" />
    </Grid>
</Window>
Примечание: XAML — это декларативный язык разметки, который описывает интерфейс приложения и его элементы.

Сравнение WPF и Windows Forms

Особенность Windows Forms WPF
Графика Ограниченная 2D графика 2D и 3D, анимации, стилизация
Разделение логики и UI Смешение кода и интерфейса Полное разделение через XAML и Code-behind
Масштабируемость Сложно поддерживать разные DPI Поддержка масштабируемой графики и векторных элементов
Стили и темы Ограниченные возможности Полная кастомизация через Styles, Templates, Triggers
Совет: Для современных приложений рекомендуется использовать WPF вместо Windows Forms, если нужна гибкая и графически насыщенная оболочка.
