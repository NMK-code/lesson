Цвет и типографика в WPF

Использование стилей и ресурсов для создания единообразного визуального оформления

Ресурсы в WPF

Ресурсы в WPF позволяют определять и повторно использовать объекты, такие как кисти, стили, шаблоны и анимации, по всему приложению.

Тип ресурса Назначение Примеры Область видимости
Локальные ресурсы Ресурсы, определенные в элементе или окне Кисти, стили для конкретного окна Только в пределах элемента и его дочерних элементов
Ресурсы приложения Глобальные ресурсы для всего приложения Цветовые схемы, общие стили Во всем приложении
Тематические ресурсы Ресурсы, определенные в темах Системные цвета, стили по умолчанию Зависит от темы ОС
Динамические ресурсы Ресурсы, которые могут изменяться во время выполнения Изменяемые цветовые схемы Во время выполнения
Статические ресурсы Ресурсы, которые определяются один раз при загрузке Фиксированные значения, константы Во время загрузки
XAML Пример определения и использования ресурсов 
<!-- Определение ресурсов -->
<Window.Resources>
    <SolidColorBrush x:Key="PrimaryBrush" Color="#FF6A11CB"/>
    <SolidColorBrush x:Key="SecondaryBrush" Color="#FF2575FC"/>
    <SolidColorBrush x:Key="AccentBrush" Color="#FF3498DB"/>
</Window.Resources>

<!-- Использование ресурсов -->
<Button 
    Background="{StaticResource PrimaryBrush}"
    Foreground="White"
    Content="Кнопка с ресурсом"/>

<Border 
    Background="{DynamicResource SecondaryBrush}"
    Padding="10">
    <TextBlock Text="Текст с динамическим ресурсом"/>
</Border>

Примечание: Используйте StaticResource для значений, которые не будут меняться во время выполнения, и DynamicResource для значений, которые могут изменяться.

Цветовые схемы и палитры

Создание согласованной цветовой схемы - ключевой аспект профессионального дизайна приложения.

Системные цвета

WPF предоставляет доступ к системным цветам, которые автоматически адаптируются под тему ОС:

XAML Пример использования системных цветов 
<Button 
    Background="{x:Static SystemColors.ControlBrush}"
    Foreground="{x:Static SystemColors.ControlTextBrush}"
    Content="Кнопка с системными цветами"/>

<Border 
    Background="{x:Static SystemColors.WindowBrush}"
    BorderBrush="{x:Static SystemColors.ActiveBorderBrush}"
    BorderThickness="1">
    <TextBlock 
        Foreground="{x:Static SystemColors.WindowTextBrush}"
        Text="Текст с системными цветами"/>
</Border>

Пользовательская цветовая палитра

Создание собственной цветовой палитры помогает поддерживать единообразие дизайна:

Primary
#6A11CB
Secondary
#2575FC
Accent
#3498DB
Dark
#2C3E50
Light
#F5F7FA
Error
#E74C3C
XAML Определение цветовой палитры в ресурсах 
<Application.Resources>
    <!-- Цвета -->
    <Color x:Key="PrimaryColor">#FF6A11CB</Color>
    <Color x:Key="SecondaryColor">#FF2575FC</Color>
    <Color x:Key="AccentColor">#FF3498DB</Color>
    
    <!-- Кисти -->
    <SolidColorBrush x:Key="PrimaryBrush" Color="{StaticResource PrimaryColor}"/>
    <SolidColorBrush x:Key="SecondaryBrush" Color="{StaticResource SecondaryColor}"/>
    <SolidColorBrush x:Key="AccentBrush" Color="{StaticResource AccentColor}"/>
    
    <!-- Градиентные кисти -->
    <LinearGradientBrush x:Key="PrimaryGradient" StartPoint="0,0" EndPoint="1,1">
        <GradientStop Color="#FF6A11CB" Offset="0"/>
        <GradientStop Color="#FF2575FC" Offset="1"/>
    </LinearGradientBrush>
</Application.Resources>

Совет: Создавайте семантически названные ресурсы (PrimaryBrush, SecondaryBrush), а не ресурсы, названные по цвету (BlueBrush, RedBrush). Это позволит легко изменять цветовую схему приложения без изменения логики.

Типографика в WPF

Единообразная типографика - важный аспект профессионального пользовательского интерфейса.

Определение стилей текста

Создание стилей для различных типов текстового контента:

Заголовок 1 (24px, Bold)
Заголовок 2 (20px, Bold)
Заголовок 3 (18px, Bold)
Обычный текст (16px, Normal)
Мелкий текст (14px, Normal)
Очень мелкий текст (12px, Normal)
XAML Определение стилей текста в ресурсах 
<Application.Resources>
    <!-- Стили типографики -->
    <Style x:Key="Header1" TargetType="TextBlock">
        <Setter Property="FontSize" Value="24"/>
        <Setter Property="FontWeight" Value="Bold"/>
        <Setter Property="Margin" Value="0,0,0,10"/>
    </Style>
    
    <Style x:Key="Header2" TargetType="TextBlock">
        <Setter Property="FontSize" Value="20"/>
        <Setter Property="FontWeight" Value="Bold"/>
        <Setter Property="Margin" Value="0,15,0,8"/>
    </Style>
    
    <Style x:Key="BodyText" TargetType="TextBlock">
        <Setter Property="FontSize" Value="14"/>
        <Setter Property="TextWrapping" Value="Wrap"/>
        <Setter Property="Margin" Value="0,0,0,5"/>
    </Style>
    
    <Style x:Key="CaptionText" TargetType="TextBlock">
        <Setter Property="FontSize" Value="12"/>
        <Setter Property="Foreground" Value="Gray"/>
        <Setter Property="FontStyle" Value="Italic"/>
    </Style>
</Application.Resources>

Использование стилей текста

XAML Пример использования текстовых стилей 
<StackPanel>
    <TextBlock Style="{StaticResource Header1}" Text="Заголовок первого уровня"/>
    <TextBlock Style="{StaticResource BodyText}" 
               Text="Это основной текст, который wraps по ширине контейнера. Он использует стиль BodyText, определенный в ресурсах приложения."/>
    
    <TextBlock Style="{StaticResource Header2}" Text="Заголовок второго уровня"/>
    <TextBlock Style="{StaticResource BodyText}" 
               Text="Еще один абзац текста, демонстрирующий использование стилей для обеспечения единообразия внешнего вида приложения."/>
    
    <TextBlock Style="{StaticResource CaptionText}" 
               Text="Подпись или дополнительная информация"/>
</StackPanel>

Примечание: Определение стилей типографики в ресурсах приложения обеспечивает единообразие внешнего вида текста во всем приложении и упрощает его изменение.

Стили элементов управления

Стили позволяют определять внешний вид элементов управления и повторно использовать эти определения.

Базовые стили кнопок

XAML Пример определения стилей кнопок 
<Application.Resources>
    <!-- Базовый стиль кнопки -->
    <Style x:Key="BaseButtonStyle" TargetType="Button">
        <Setter Property="Padding" Value="12,6"/>
        <Setter Property="Margin" Value="5"/>
        <Setter Property="Foreground" Value="White"/>
        <Setter Property="BorderThickness" Value="1"/>
        <Setter Property="FontWeight" Value="SemiBold"/>
        <Setter Property="Cursor" Value="Hand"/>
    </Style>
    
    <!-- Основная кнопка -->
    <Style x:Key="PrimaryButtonStyle" 
           TargetType="Button" 
           BasedOn="{StaticResource BaseButtonStyle}">
        <Setter Property="Background" Value="{StaticResource PrimaryBrush}"/>
        <Setter Property="BorderBrush" Value="{StaticResource PrimaryBrush}"/>
    </Style>
    
    <!-- Вторичная кнопка -->
    <Style x:Key="SecondaryButtonStyle" 
           TargetType="Button" 
           BasedOn="{StaticResource BaseButtonStyle}">
        <Setter Property="Background" Value="Transparent"/>
        <Setter Property="Foreground" Value="{StaticResource PrimaryBrush}"/>
        <Setter Property="BorderBrush" Value="{StaticResource PrimaryBrush}"/>
    </Style>
    
    <!-- Стиль по умолчанию для всех кнопок -->
    <Style TargetType="Button" BasedOn="{StaticResource BaseButtonStyle}"/>
</Application.Resources>

Триггеры в стилях

Триггеры позволяют изменять внешний вид элементов в ответ на различные события и состояния:

XAML Пример использования триггеров в стилях 
<Style x:Key="ModernButtonStyle" TargetType="Button" 
       BasedOn="{StaticResource BaseButtonStyle}">
    <Setter Property="Background" Value="{StaticResource PrimaryBrush}"/>
    <Setter Property="BorderBrush" Value="Transparent"/>
    <Setter Property="Template">
        <Setter.Value>
            <ControlTemplate TargetType="Button">
                <Border x:Name="border" 
                        Background="{TemplateBinding Background}"
                        BorderBrush="{TemplateBinding BorderBrush}"
                        BorderThickness="{TemplateBinding BorderThickness}"
                        CornerRadius="4">
                    <ContentPresenter HorizontalAlignment="Center"
                                      VerticalAlignment="Center"
                                      Margin="{TemplateBinding Padding}"/>
                </Border>
                <ControlTemplate.Triggers>
                    <Trigger Property="IsMouseOver" Value="True">
                        <Setter TargetName="border" Property="Background" 
                               Value="{StaticResource SecondaryBrush}"/>
                    </Trigger>
                    <Trigger Property="IsPressed" Value="True">
                        <Setter TargetName="border" Property="Background" 
                               Value="{StaticResource AccentBrush}"/>
                    </Trigger>
                    <Trigger Property="IsEnabled" Value="False">
                        <Setter TargetName="border" Property="Opacity" Value="0.6"/>
                    </Trigger>
                </ControlTemplate.Triggers>
            </ControlTemplate>
        </Setter.Value>
    </Setter>
</Style>

Совет: Используйте свойство BasedOn для наследования стилей. Это позволяет создавать базовый стиль и затем расширять его для конкретных случаев, уменьшая дублирование кода.

Темы и скины

WPF поддерживает создание сменных тем через механизм ResourceDictionary:

XAML Пример переключения тем 
<!-- LightTheme.xaml -->
<ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Color x:Key="BackgroundColor">#FFFFFFFF</Color>
    <Color x:Key="ForegroundColor">#FF333333</Color>
    <SolidColorBrush x:Key="BackgroundBrush" Color="{StaticResource BackgroundColor}"/>
    <SolidColorBrush x:Key="ForegroundBrush" Color="{StaticResource ForegroundColor}"/>
</ResourceDictionary>

<!-- DarkTheme.xaml -->
<ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <Color x:Key="BackgroundColor">#FF333333</Color>
    <Color x:Key="ForegroundColor">#FFFFFFFF</Color>
    <SolidColorBrush x:Key="BackgroundBrush" Color="{StaticResource BackgroundColor}"/>
    <SolidColorBrush x:Key="ForegroundBrush" Color="{StaticResource ForegroundColor}"/>
</ResourceDictionary>

<!-- Код для переключения тем -->
<Button Content="Светлая тема" 
        Click="LightTheme_Click"/>
<Button Content="Темная тема" 
        Click="DarkTheme_Click"/>
C# Обработчики переключения тем 
private void LightTheme_Click(object sender, RoutedEventArgs e)
{
    // Загружаем светлую тему
    Application.Current.Resources.MergedDictionaries.Clear();
    Application.Current.Resources.MergedDictionaries.Add(
        new ResourceDictionary() { Source = new Uri("LightTheme.xaml", UriKind.Relative) });
}

private void DarkTheme_Click(object sender, RoutedEventArgs e)
{
    // Загружаем темную тему
    Application.Current.Resources.MergedDictionaries.Clear();
    Application.Current.Resources.MergedDictionaries.Add(
        new ResourceDictionary() { Source = new Uri("DarkTheme.xaml", UriKind.Relative) });
}

Примечание: Для динамического переключения тем используйте DynamicResource вместо StaticResource, чтобы изменения применялись без перезагрузки приложения.
